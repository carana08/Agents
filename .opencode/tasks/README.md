# Tasks Directory

Purpose: define execution-level tasks that map accepted feature scope to explicit subagent ownership.

Rules:
- Each task file must be linked to one active feature.
- Each task file must assign one primary subagent.
- A task must not redefine mission, constitution, or tech-stack policy.
- Dependency changes inside a task require explicit approval.
- A task is complete only when validation evidence is recorded.

Recommended flow:
1. Create a task from an accepted feature.
2. Assign primary subagent and supporting subagents.
3. Execute checklist steps.
4. Record validation evidence.
5. Mark task status.
