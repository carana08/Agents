# Research

Purpose of `research/`: store technical decision records used during planning before implementation.

## Rules
- Do not use `research/` to redefine the mission, roadmap, or global project rules.
- Use `research/` to document technical questions, evaluated options, findings, decisions, and implications.
- Each research file must be question-driven and decision-oriented.
- Treat accepted research files as decision-support artifacts for agents and skills.
- Do not contradict accepted research decisions without explicit justification.

## Required Structure
Each research file should end with:
- context
- question
- options considered
- evaluation criteria
- findings
- decision
- rationale
- constraints
- implications
- validation
- status
